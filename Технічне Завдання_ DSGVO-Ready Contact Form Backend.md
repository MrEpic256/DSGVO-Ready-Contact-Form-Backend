# Технічне Завдання: DSGVO-Ready Contact Form Backend

## 1. Огляд Проекту
**Назва:** DSGVO-Ready Contact Form Backend
**Ціль:** Розробити надійний бекенд-сервіс, який безпечно обробляє дані контактних форм, суворо дотримуючись вимог Загального регламенту про захист даних (DSGVO/GDPR).
**Демонстраційна мета:** Показати здатність створювати **правово-безпечні (rechtssicher)** рішення для німецьких клієнтів, які є надзвичайно чутливими до питань захисту даних.

## 2. Технічний Стек
*   **Бекенд:** TypeScript (Node.js/Express) або Python (FastAPI).
*   **База даних:** PostgreSQL (або інша реляційна БД).
*   **Контейнеризація:** Docker.

## 3. Ключові Функціональні Вимоги

### 3.1. API для Обробки Запитів
1.  **Ендпоінт:** Створення єдиного POST-ендпоінту `/api/v1/contact/submit`.
2.  **Валідація Введення:** Обов'язкова валідація полів: `name`, `email`, `message`, `consent_checkbox` (булеве значення).
3.  **Перевірка Згоди (Consent):** Запит повинен бути відхилений (HTTP 400 Bad Request), якщо `consent_checkbox` не встановлено.

### 3.2. Обробка та Зберігання Даних
1.  **Мінімалізація Даних:** Зберігати лише ті дані, які були надані користувачем (ім'я, email, повідомлення, час відправки).
2.  **Анонімізація IP:** Перед збереженням IP-адреси відправника, необхідно її анонімізувати, обрізавши останній октет (наприклад, `192.168.1.1` стає `192.168.1.0`). Зберігати анонімізовану IP-адресу та User-Agent для цілей аудиту.
3.  **Захист даних:** Використання параметризованих запитів для запобігання SQL-ін'єкціям.

### 3.3. Реалізація "Права на Забуття" (Recht auf Vergessenwerden)
1.  **Ендпоінт Видалення (Адмін):** Створення захищеного ендпоінту (наприклад, з простим ключем доступу) `/api/v1/contact/delete/{email}`, який повністю видаляє всі записи, пов'язані з вказаним email.
2.  **Автоматичне Видалення (Опціонально):** Впровадження простого механізму, який автоматично видаляє записи старше 6 місяців (імітація політики зберігання даних).

## 4. Нефункціональні Вимоги
*   **Безпека:** Використання HTTPS (імітація в документації), санітизація вхідних даних.
*   **Документація:** Детальний `README.md`, що описує політику зберігання даних та кроки, вжиті для забезпечення відповідності DSGVO.
